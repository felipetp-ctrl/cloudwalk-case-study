# ML-Based Anomaly Detection for Edge HTTP Traffic

An ML pipeline that classifies incoming HTTP request sequences as malicious or benign in real time at the edge, enabling automated blocking before requests reach origin servers. Built for CloudWalk's edge infrastructure constraints: <5ms inference, WASM runtime, no GPU.

See **[WRITEUP.md](WRITEUP.md)** for the full technical writeup covering reasoning, results, and deep-dive answers.

## Prerequisites

- **Python 3.10+**
- **Rust toolchain** — only needed for the edge-inference benchmark (`rustup` — https://rustup.rs)
- **wasmtime** — only needed for the WASM benchmark (`brew install wasmtime` on macOS)

## Running

### 1. Python Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Register the kernel so Jupyter can find it:

```bash
python -m ipykernel install --user --name cloudwalk --display-name "CloudWalk"
```

### 2. Notebooks (run in order)

Each notebook depends on the output of the previous one. Run them sequentially:

```bash
jupyter notebook
```

| Order | Notebook | What it does | What it produces |
|---|---|---|---|
| 1 | `notebooks/eda.ipynb` | Joins incident labels to requests (IP, CIDR, TLS + temporal bounds). Exploratory analysis of attack patterns. | Labeled dataset with 592 malicious / 49,408 benign requests |
| 2 | `notebooks/feature_engineering.ipynb` | Computes 15 per-request + 78 session features with causal windowing. Analyzes feature distributions. | Feature matrix ready for modeling |
| 3 | `notebooks/baseline_model.ipynb` | Trains 4 models (LR, RF, XGB, LGBM) with Optuna tuning. Evaluates on temporal split. Pruning and threshold analysis. | `plots/pr_curves.png`, `plots/feature_importance.png`, `plots/cost_curve.png`, trained LGBM model |
| 4 | `notebooks/edge_deployment.ipynb` | Exports LGBM to ONNX. Benchmarks Python inference (LightGBM native vs ONNX Runtime). Prepares model for Rust/WASM benchmark. | `edge-inference/model/model_full.onnx`, `edge-inference/model/sample_input.json` |

### 3. Tests

Run the full test suite from the project root:

```bash
pytest -v
```

Expected output: **39 tests passed** (with sklearn FutureWarning deprecation notices — non-critical).

Individual test modules:

```bash
pytest tests/test_label_joining.py -v   # 4 tests — label joining strategies
pytest tests/test_features.py -v        # 9 tests — feature computation, causal windowing
pytest tests/test_model.py -v           # 21 tests — training, evaluation, tuning, pruning
pytest tests/test_export.py -v          # 2 tests — ONNX export and numerical validation
pytest tests/test_pipeline.py -v        # 2 tests — end-to-end pipeline
```

### 4. Edge Inference Benchmark (Rust/WASM)

Requires notebook 4 to have run first (generates the ONNX model).

```bash
cd edge-inference
```

**Native Rust benchmark:**

```bash
rustup target add wasm32-wasip1    # one-time setup
cargo build --release
cargo run --release
```

**WASM benchmark:**

```bash
cargo build --release --target wasm32-wasip1
wasmtime run --dir model target/wasm32-wasip1/release/edge-inference.wasm
```

Both benchmarks report p50/p95/p99 latency and throughput per thread. Expected results: p99 < 1ms for both native and WASM (well within the 5ms budget).

### 5. Using the Pipeline Programmatically

The `src/` modules can be used independently:

```python
from src.pipeline import build_training_dataset

df = build_training_dataset(
    requests_path="data/http_requests.csv",
    headers_path="data/request_headers.csv",
    labels_path="data/incident_labels.csv",
)
# df contains 57 features + labels + sample_weight, ready for modeling
```
